# PMOS 滑动验证码处理模块 - 技术说明

## 一、背景信息

网页滑块验证码通常由两张图组成：
1. **带缺口的背景图** - 完整的图片中有一个拼图缺口
2. **滑块图** - 长方形图片，高度与背景图相同，宽度较窄，包含拼图块的形状（带透明通道）

用户的任务是将滑块拖动到背景图的缺口位置，使两者完美吻合。

## 二、核心解决思路

### 2.1 二维边缘轮廓形状匹配

程序采用**二维边缘轮廓交叉互相关匹配**算法，核心思想是：

```
滑块边缘形状 ⊗ 背景边缘 = 最佳匹配位置
```

这种方法能够识别同一水平线上存在的多个假缺口，精准找到真实缺口。

### 2.2 技术路线

```
输入图片
    ↓
┌─────────────────────────────────────┐
│ 第一步：提取滑块真实形状             │
│ - 解析RGBA透明通道                   │
│ - 获取拼图块的精确Y坐标和X偏移       │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 第二步：截取背景图的水平带           │
│ - 根据滑块Y坐标，上下各加5像素容差   │
│ - 只处理相关区域，减少干扰           │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 第三步：边缘检测                     │
│ - 高斯模糊去噪声                     │
│ - Canny算子提取边缘                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 第四步：消除滑块外围黑边干扰         │
│ - 利用Alpha透明通道创建掩膜          │
│ - 腐蚀操作去除抗锯齿脏边缘           │
│ - 只保留真正的拼图曲线轮廓           │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 第五步：模板匹配                     │
│ - 使用TM_CCOEFF_NORMED归一化相关     │
│ - 对光照、对比度高度鲁棒             │
│ - 只看形状吻合度                     │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 第六步：排除左侧初始坑位             │
│ - 验证码最左侧常留有滑块初始坑       │
│ - 强制将左侧匹配得分置为极低         │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 第七步：距离补偿                     │
│ - 阴影补偿：+8px（背景缺口左边缘有暗色阴影） │
│ - 腐蚀补偿：+像素（腐蚀让轮廓变瘦了） │
│ - 初始边距补偿：滑块左侧透明空白     │
└─────────────────────────────────────┘
    ↓
输出：缺口X坐标、Y坐标、置信度
```

### 2.3 关键技术点

| 技术点 | 说明 | 作用 |
|--------|------|------|
| RGBA透明通道解析 | 提取滑块的真实拼图形状 | 排除方形边框干扰 |
| Y轴水平带截取 | 只处理滑块所在水平区域 | 减少其他边缘干扰 |
| Canny边缘检测 | 提取轮廓信息 | 获取形状特征 |
| Alpha掩膜腐蚀 | 去除抗锯齿产生的脏边缘 | 获得干净轮廓 |
| TM_CCOEFF_NORMED | 归一化互相关匹配 | 对光照变化鲁棒 |
| 左侧坑位屏蔽 | 排除初始位置的假缺口 | 避免误匹配 |
| 自适应补偿 | 根据历史结果动态调整补偿值 | 提高成功率 |

### 2.4 拟人化拖动轨迹

程序模拟人类真实拖动行为：
- **起步阶段**：慢加速
- **中间阶段**：快速且稳定
- **接近目标**：明显减速
- **精确定位**：可能有过冲和回调
- **随机波动**：速度和位置有轻微随机抖动

使用 **easeInOutCubic** 缓动曲线生成自然轨迹。

### 2.5 自适应学习机制

程序会记录每次验证的结果（使用的补偿值、成功/失败），通过加权平均动态调整最优补偿值：

```python
new_offset = Σ(历史补偿值 × 权重) / Σ权重
```

权重按时间衰减，最近的记录权重更高。

## 三、完整处理流程

```python
# 1. 查找滑块元素
slider_element = find_slider_element(page)

# 2. 获取验证码图片（背景图 + 滑块图）
bg_image, block_image = get_captcha_images(page, container)

# 3. 识别缺口位置
gap_x, gap_y, confidence = captcha_solver.get_slide_distance(bg_image, block_image)

# 4. 计算滑动距离（考虑图片偏移、补偿等）
distance = gap_x_on_screen - slider_x + offset

# 5. 生成拟人化轨迹
tracks = generate_drag_tracks(distance)

# 6. 模拟鼠标拖动
simulate_drag(page, slider_element, tracks)

# 7. 检查结果，失败则调整补偿值重试
```

## 四、算法优势

1. **专杀同水平多假缺口** - 二维形状匹配不怕多个假缺口
2. **光照鲁棒性强** - 归一化相关系数对光照变化不敏感
3. **位置精准** - 边缘匹配 + 多重补偿，精度达到像素级
4. **行为拟人** - 轨迹自然，难以被检测为机器人
5. **自我优化** - 自适应学习机制，越用越准
